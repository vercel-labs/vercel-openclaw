## Design Context

### Users
Solo developers managing their own OpenClaw sandbox instance on Vercel. They arrive at the admin UI to check status, restore sandboxes, configure channels, and manage firewall rules. They're technical, comfortable with CLIs, and value speed and clarity over hand-holding.

### Brand Personality
**Technical, Reliable, Minimal** — an engineering tool that stays out of the way. It should feel like a natural extension of the Vercel ecosystem: precise, confident, and quiet.

### Aesthetic Direction
- **Visual tone**: Dark-first, typographic, information-dense but not cluttered. Generous whitespace separates logical groups.
- **References**: Vercel Dashboard (minimal, monospace accents, restrained use of color), Linear (dense but polished, subtle motion, keyboard-first).
- **Anti-references**: Overly colorful dashboards, playful/rounded UI, anything that feels like a consumer product or marketing page.
- **Theme**: Dark mode only. Pure black (`#000`) background, `#ededed` foreground, semantic status colors used sparingly for signal.

### Design Principles

1. **Signal over decoration** — Every visual element must carry information. No ornamental gradients, shadows, or illustrations. Color is reserved for status and brand.
2. **Density with clarity** — Pack information tight but keep it scannable. Use Geist Mono eyebrow labels, consistent spacing scales, and strong typographic hierarchy to organize dense panels.
3. **Quiet confidence** — The interface should feel stable and trustworthy. Subtle transitions (150ms ease), restrained animations, no jarring motion. Status changes communicate through color and text, not flashy effects.
4. **Vercel-native feel** — Align with Vercel's design language: Geist fonts, monochrome palette, 1px borders, pill badges, and the same component vocabulary (cards, tabs, badges, ghost buttons).
5. **Operator-first** — Optimize for the person who already knows what everything means. Favor concise labels and direct controls over explanatory UI. The terminal tab and CLI-copy patterns are first-class.

### Design Tokens (Reference)

| Token | Value | Usage |
|-------|-------|-------|
| `--background` | `#000` | Primary surface |
| `--background-elevated` | `#0a0a0a` | Cards, elevated panels |
| `--foreground` | `#ededed` | Primary text |
| `--foreground-muted` | `#888` | Secondary text |
| `--foreground-subtle` | `#666` | Tertiary text, labels |
| `--border` | `rgba(255,255,255,0.08)` | Default borders |
| `--border-strong` | `rgba(255,255,255,0.14)` | Emphasized borders |

### Typography Scale

- **h1**: `clamp(1.75rem, 3.5vw, 2.5rem)`, weight 700, line-height 1.1
- **h2**: `1.25rem`, weight 600, line-height 1.2
- **Eyebrow**: Geist Mono 11px, weight 500, uppercase, `0.06em` tracking
- **Body**: `0.875rem`, line-height 1.5
- **Code**: Geist Mono `12–13px`

### Component Vocabulary

Buttons (primary/secondary/ghost/danger/success), panel cards, hero cards, status badges with animated dots, channel pills, tab navigation, text inputs, field stacks, domain lists, event rows, log rows. No external component library — all custom CSS with BEM-inspired naming and CSS custom properties.

### Accessibility

WCAG AA compliance. Focus rings (`2px solid var(--ring)`), semantic HTML, ARIA attributes on interactive components, keyboard navigable tabs and controls.
